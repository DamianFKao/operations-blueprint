# Operations system (starter)

Generated by the [Operations Blueprint](https://damiankao.com/blueprint) for a metal fabrication operation.
A foundation-first build: a relational core, an API that is the only writer, and everything priced
through one engine.

## Run

1. Copy `.env.example` to `.env`.
2. `docker compose up` (starts Postgres with `db/schema.sql` loaded, and the API on :8000).
3. Open `http://localhost:8000/health`.

## Build

Read `AGENTS.md`, then work through `TASKS.md` in order. Start with the cost engine
(`api/engine/cost.py`): it is the keystone everything else prices through.

## Layout

```
db/schema.sql      the relational source of truth (Postgres)
api/engine/cost.py the cost engine (build first)
api/routes/        HTTP surfaces, one per domain
AGENTS.md          rules and build order for the agent
TASKS.md           the build checklist
docs/blueprint.md  the full plan
```

The full plan, including the data model and the integrator notes, is in `docs/blueprint.md`.
